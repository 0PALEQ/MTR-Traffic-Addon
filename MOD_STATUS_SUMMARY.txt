Current feature status
======================

- Built-in sedan traffic vehicles render through the custom OBJ traffic model path and remain available as embedded resources.
- Traffic dashboard controls for connector state, intersection state, signal modes, and vehicle pools remain present.
- Traffic spawn/despawn connectors still use MTR rail geometry and saved connector metadata.
- Traffic light blocks, pole attachment models, glow overlays, and brush binding behavior are implemented.
- Manual and Auto intersection signal modes are implemented.
- MTR blocker checks fail open when addon traffic state is stale, so paused or unloaded addon state should not indefinitely block MTR vehicles.
- Traffic simulation now runs on a dedicated wall-clock simulation thread instead of moving addon vehicles directly in the Minecraft server tick path.
- Spawn connectors now create deterministic virtual route streams globally. Vehicles are materialized only when their calculated route position is near a player.
- Vehicle rendering and materialization are distance limited by config. Defaults are automatic:
  - visibility distance = render distance - 2 chunks
  - simulation distance = visibility distance + materialization margin
  - materialization margin = 2 chunks
  - unrendered active vehicles are removed after 30 seconds
- Active addon vehicles outside every player's simulation distance are removed and can be recreated later from the virtual stream.
- MTR vehicles ignore addon intersection lights when no player is close enough for that intersection to be actively simulated.
- MTR intersection demand detection now uses route lookahead, so MTR vehicles stopped before an entry rail can still queue the correct intersection input group.

Unfinished / hidden features
============================

- MTA Traffic Connector items are unfinished.
- They remain registered so old worlds, old saves, and existing item stacks do not break.
- They are hidden from the addon creative tab.
- Their display names are marked with "(Unfinished)".
- Creating new MTA-only connector tracks is disabled in normal builds.
- Existing saved MTA-only connector tracks can still be removed by reconnecting the same two nodes with an existing MTA Traffic Connector item, or by using the standard MTR Rail Remover.
- There is a JVM flag for local development only:
  -Dmta.enableUnfinishedTrafficConnectors=true

26.5.B02 changelog
==================

Performance and simulation
--------------------------

- Moved addon traffic vehicle movement, spacing, signal decisions, materialization, and despawn checks to a dedicated wall-clock simulation loop.
- Kept Minecraft server ticks for player snapshots, graph refresh snapshots, and networking.
- Added automatic distance configuration in:
  config/mtr-traffic-addon.properties
- Added:
  trafficVehicleVisibilityDistanceBlocks
  trafficVehicleSimulationDistanceBlocks
  trafficVehicleMaterializationMarginChunks
  trafficVehicleUnrenderedLifetimeSeconds
- Default visibility now follows player/server render distance instead of rendering all addon vehicles from long routes.
- Default simulation distance follows visibility distance plus a margin so vehicles can exist before entering view.
- Added virtual route streams so spawn points can keep simulated departure timing without spawning thousands of active world vehicles across a large network.
- Added removal of active addon vehicles that leave all players' simulation ranges.
- Added removal of active addon vehicles that have not been sent/rendered for the configured timeout.
- Prevented virtual vehicles from materializing on spawn/despawn connector segments or their immediately adjacent route segments.

MTR compatibility and intersections
-----------------------------------

- Added fail-open behavior for addon intersection lights when an intersection is outside every player's active simulation radius.
- Added MTR route lookahead tracking for intersection demand.
- Fixed cases where MTR vehicles stopped before an intersection entry rail did not add their entry track to the intersection queue.
- Kept stale traffic blocker behavior fail-open so old addon state does not keep MTR vehicles stopped indefinitely.

Traffic networking and rendering
--------------------------------

- Server-to-client traffic snapshots now respect visibility distance before sending active addon vehicles.
- The client renderer also checks local visibility distance and loaded chunks before drawing addon vehicles.
- Rendered vehicles are reported back to the server so unrendered active vehicles can expire after the configured timeout.

MTA-only connector experiment
-----------------------------

- Added internal support for addon-exclusive rail connector definitions and world storage:
  data/mtr-traffic-addon/mta_exclusive_rails.json
- Added internal client sync and renderer support for those connector definitions.
- Added MTR Rail Remover compatibility for removing matching MTA-only connector definitions.
- Marked the feature unfinished after testing showed it is not reliable enough for normal players.
- Hid the connector items from the creative tab and disabled new connector creation in normal builds.
- Kept item IDs, models, textures, and saved-data loading for compatibility.

Documentation
-------------

- Updated README with current distance/simulation behavior and unfinished MTA Traffic Connector status.
- Updated docs/ADDON_DOCUMENTATION.md with virtual traffic behavior, distance config, unrendered vehicle timeout, intersection simulation behavior, and unfinished connector status.

Known notes and remaining work
==============================

- The default Java on this machine may point to Java 17; set JAVA_HOME to Java 21+ before release builds because Fabric Loom requires it.
- Large-network traffic behavior should be tested in-game on a real server with multiple players and long route networks.
- The wall-clock simulation loop is new and should be watched for thread-safety regressions around MTR vehicle occupancy, intersections, and dashboard snapshots.
- Auto intersections still depend on graph snapshots and recent vehicle observations near players.
- MTR-only remote intersections intentionally fail open when no player is nearby; this is a performance tradeoff.
- Spawn density controls are still basic and are based on spawn interval plus maxVehicles.
- maxVehicles limits recent virtual departures, but dashboard controls for changing it are still not exposed.
- Traffic still depends on MTR rail geometry, so road quality depends on the underlying rail graph.
- MTA Traffic Connectors should stay hidden/disabled until the routing, rendering, node behavior, and player workflow are redesigned and tested.
- Existing worlds that already contain saved MTA-only connector definitions may keep those definitions in saved data until removed.
- Traffic light glow placement should still be checked in-game from all four facings after model or texture changes.
