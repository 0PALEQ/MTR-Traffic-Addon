Current feature status
======================

- Built-in sedan and hatchback traffic vehicles render through the custom OBJ traffic model path and remain available as embedded resources.
- The built-in sedan vehicle set includes the taxi variant:
  mta_sedan_taxi / sedan_taxi
- The built-in hatchback vehicle set includes nine visual/resource variants:
  mta_hatchback, mta_hatchback_white, mta_hatchback_gray, mta_hatchback_blue, mta_hatchback_brown, mta_hatchback_green, mta_hatchback_orange, mta_hatchback_pink, mta_hatchback_red
- OBJ custom traffic models support .mtl material libraries, usemtl material selection, and map_Kd per-face texture selection. Faces without material textures use the model definition default texture.
- Traffic vehicles now carry route pitch to the client, and render with a small deterministic pitch variation plus deterministic lateral offset so vehicles do not overlap as perfectly on the same route centerline.
- Traffic dashboard controls for connector state, intersection state, signal modes, and vehicle pools remain present.
- Traffic spawn/despawn connectors still use MTR rail geometry and saved connector metadata.
- Traffic light blocks, pole attachment models, glow overlays, and brush binding behavior are implemented.
- Pedestrian Lights and Pedestrian Lights Pole blocks are implemented, exposed in the creative tab, mineable with pickaxe, and bindable with the MTR brush.
- Traffic light bindings can target either an intersection IN node or a signal group. Group bindings are intended for pedestrian phases.
- Manual and Auto intersection signal modes are implemented.
- MTR blocker checks fail open when addon traffic state is stale, so paused or unloaded addon state should not indefinitely block MTR vehicles.
- Traffic simulation runs on a dedicated wall-clock simulation thread instead of moving addon vehicles directly in the Minecraft server tick path.
- Spawn connectors create deterministic virtual route streams globally. Vehicles are materialized only when their calculated route position is near a player and has clearance from active addon/MTR vehicles.
- Vehicle rendering and materialization are distance limited by config. Defaults are automatic:
  - visibility distance = render distance - 2 chunks
  - simulation distance = visibility distance + materialization margin
  - materialization margin = 2 chunks
  - unrendered active vehicles are removed after 30 seconds
- Active addon vehicles outside every player's simulation distance are removed and can be recreated later from the virtual stream.
- MTR vehicles ignore addon intersection lights when no player is close enough for that intersection to be actively simulated.
- MTR intersection demand detection uses route lookahead, so MTR vehicles stopped before an entry rail can still queue the correct intersection input group.

Unfinished / hidden features
============================

- MTA Traffic Connector items are unfinished.
- They remain registered so old worlds, old saves, and existing item stacks do not break.
- They are hidden from the addon creative tab.
- Their display names are marked with "(Unfinished)".
- Creating new MTA-only connector tracks is disabled in normal builds.
- Existing saved MTA-only connector tracks can still be removed by reconnecting the same two nodes with an existing MTA Traffic Connector item, or by using the standard MTR Rail Remover.
- There is a JVM flag for local development only:
  -Dmta.enableUnfinishedTrafficConnectors=true

Traffic lights and bindings
---------------------------

- Added Pedestrian Lights and Pedestrian Lights Pole blocks, items, blockstates, models, textures, loot tables, translations, creative-tab entries, pickaxe tags, and emissive overlays.
- Added TrafficLightBindingTargetType with NODE and GROUP target modes.
- Updated saved traffic light bindings to retain backwards compatibility with old node-only bindings while supporting group targets.
- Updated the brush binding screen/networking to list signal groups before IN nodes and send target type plus target number.
- Added group signal-state lookup so bound lights can follow a whole signal group.

Simulation and spacing
----------------------

- Added materialization clearance checks against active addon vehicles and recently observed MTR vehicles.
- Added skipped virtual vehicle tracking so unsafe virtual departures are not retried every simulation pass.
- Added next-segment entry limits to prevent vehicles entering occupied/full segments.
- Refined signal occupancy checks to apply at signal entry boundaries instead of every segment that shares a signal color.
- Spawn connector route traversal now prefers saved node direction, while retaining reverse traversal for old saved spawn points.
- Despawn connector segments remain protected from vehicle materialization.

Known notes and remaining work
==============================

- Set JAVA_HOME to Java 21+ before release builds because Fabric Loom requires it.
- Large-network traffic behavior should be tested in-game on a real server with multiple players and long route networks.
- The wall-clock simulation loop should still be watched for thread-safety regressions around MTR vehicle occupancy, intersections, dashboard snapshots, and materialization.
- Auto intersections still depend on graph snapshots and recent vehicle observations near players.
- MTR-only remote intersections intentionally fail open when no player is nearby; this is a performance tradeoff.
- Spawn density controls are still basic and are based on spawn interval plus maxVehicles.
- maxVehicles limits recent virtual departures, but dashboard controls for changing it are still not exposed.
- Traffic still depends on MTR rail geometry, so road quality depends on the underlying rail graph.
- MTA Traffic Connectors should stay hidden/disabled until the routing, rendering, node behavior, and player workflow are redesigned and tested.
- Existing worlds that already contain saved MTA-only connector definitions may keep those definitions in saved data until removed.
- Pedestrian lights currently use red/green pedestrian indications only. Yellow group state has no pedestrian glow overlay.
- Traffic light glow placement should still be checked in-game from all four facings after model or texture changes.
