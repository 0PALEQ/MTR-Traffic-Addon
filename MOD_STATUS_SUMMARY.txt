Traffic light binding and signal behavior
=========================================

- Added brush interaction support for traffic lights.
- Right-clicking a traffic light with the MTR brush inside an intersection opens a node selection screen.
- The selection screen lists valid intersection node IDs for binding a traffic light.
- OUT nodes are filtered out and cannot be selected for traffic light binding.
- Bound traffic lights read the current intersection signal state and update their blockstate/model to match:
  - red
  - yellow
  - green
  - off
- Added the same binding behavior for traffic lights attached to vertical poles.
- Fixed yellow signal behavior so only the node/group currently transitioning becomes yellow, instead of every node turning yellow.

Traffic light rendering fixes
=============================

- Added client-side emissive overlay rendering for lit traffic light lenses.
- Added glow overlays for red, yellow, and green states.
- Fixed black/opaque invisible model faces by putting traffic light blocks on the cutout render layer.
- Fixed earlier issues where the entire traffic light model became emissive instead of only the active lens.
- Fixed several incorrect glow overlay placements.
- Latest fix changes the emissive overlay rotation for east/west facing models so the glowing element renders on the front of the traffic light instead of the back.

26.5.A04 changes since 26.5.A03
===============================

- Updated project version output from 26.5.A03 to 26.5.A04.
- Added traffic light signal modes:
  - Manual
  - Auto
- Manual mode keeps the previous phase-order based traffic light behavior.
- Auto mode detects waiting vehicles before IN nodes assigned to an intersection.
- Auto mode queues the group containing the demanded IN node.
- Auto mode switches the currently green group to red after a delay when another group is waiting.
- Auto mode waits for the intersection to clear before giving green to the next queued group.
- Auto mode keeps the current group green while no other group has demand.
- Green light duration now has a minimum of 15 seconds.
- A green phase can end early if the current group has no cars left and another demanded group is queued.
- Yellow/all-red clearance is applied before giving green to the next group, even when the current group is already empty.
- Fixed unrealistic instant green-to-green changes when an empty group hands off to another group.
- Fixed a case where vehicles could stop permanently because a vehicle standing near an IN node or stop line was treated as blocking intersection clearance forever.
- Added dashboard/network snapshot support for the signal mode value.
- Added dashboard controls for switching an intersection between Manual and Auto.
- Updated intersection signal timing text and dashboard state to reflect the newer signal behavior.
- Added a signal clock based on wall time/server tick synchronization so signal checks remain stable during pauses or tick gaps.
- Limited stale MTA traffic vehicle obstacle checks after long pauses so old positions do not indefinitely block MTR route vehicles.

Known notes and remaining work
==============================

- The default Java on this machine currently points to Java 17; set JAVA_HOME to Java 21+ before release builds because Fabric Loom requires Java 21+.
- Auto signal behavior has been tested interactively, but larger intersections with many groups should still be tested in-game.
- The new sedan mesh now renders correctly in-game after the OBJ renderer fixes, but other future mesh formats may need separate loader support.
- The external resourcepack copy should not be enabled alongside the mod jar unless intentionally overriding embedded resources.
- Existing worlds or spawn connector pools that referenced removed/old sedan IDs should be checked and refreshed if needed.
- Traffic light phase mechanics are intentionally still basic; more model/state variants are expected before implementing richer light-changing behavior.
- Traffic light glow placement should still be checked in-game from all four facings after the latest east/west overlay rotation fix.
